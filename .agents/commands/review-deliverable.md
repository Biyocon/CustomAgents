# Command: Review dette

> Udfør teknisk QA-review af en leverance
> Trigger: "Review dette" / "Gennemgå denne leverance"

## Workflow

1. **Forstå leverancen**
   - Hvad er formålet?
   - Hvem er modtageren?
   - Hvilke krav skal opfyldes?

2. **Tjek faglig kvalitet**
   - Korrekte antagelser?
   - Konsistent terminologi?
   - Passende detaljeringsgrad?
   - Tydelige åbne punkter?

3. **Tjek compliance**
   - Overholder den relevante standarder/regler?
   - Er BDK/BBTR/BBE/BKP-krav respekteret (hvis relevant)?

4. **Dokumentér fund**
   - Notér i `brain/personal/lessons-learned.md`
   - Opret QA-poster i `templates/qa/`

5. **Giv anbefaling**
   - Godkend / godkend med rettelser / afvis med begrundelse
