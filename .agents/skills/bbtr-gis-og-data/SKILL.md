# GIS og data

## Formål
At understøtte GIS- og datamanagement-opgaver i Banedanmark-kontekst: kortproduktion, vedligeholdelse af geografisk infrastrukturdata, geografisk modellering af trafikale data, dataudtræk til interne og eksterne interessenter samt datakoordinering — med korrekt ansvarsplacering og forsvarlig datahåndtering.

## Fremgangsmåde
1. Afklar opgavetypen: kortproduktion (rapporter, projekter, websider), webGIS-applikationer, dataudtræk, vedligehold af geografisk infrastruktur eller geografisk modellering af trafikale data — jf. ansvarsområderne i FB'en "GIS-specialist.pdf".
2. Identificér ansvarlig rolle og beføjelser via FB'erne (GIS-specialist har fx beføjelsen at "beslutte design af kortlayout"; Data Manager evaluerer og håndterer informationsanmodninger fra leverandører). Brug **bdk-roller-kompetencer** til struktureret uddrag.
3. Afklar datakilder og autoritative registre for opgaven (FB'en nævner fx adresseudtræk fra OIS og eksport af GIS-data); dokumentér hvilken kilde der er autoritativ for hvert datalag, og markér usikre kilder eksplicit.
4. Ved udtræk og deling af data: vurdér modtager (intern/ekstern), persondataindhold (**bdk-gdpr-praksis**) og evt. klassifikation af infrastrukturdata; del kun med relevante parter jf. adgangsprincippet i **bbtr-dokumentstyring**.
5. Versionér datasæt og kortprodukter: angiv dataudtræksdato, kildeversion og ansvarlig i metadata, så leverancen er reproducerbar og sporbar.
6. Ved leverancer til BBTR-projekter: koordinér med projekterings- og trafikroller så geodata-grundlaget matcher projektets fase og lokation, og registrér leverancen i projektets dokumentliste jf. **bbtr-dokumentstyring**.
