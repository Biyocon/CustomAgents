# Hændelseshåndtering i BBTR-projekter

## Formål
At sikre, at hændelser, nærved-hændelser og sikkerhedsafvigelser i BBTR-projekter håndteres ensartet: akut reaktion, korrekt klassifikation og indmelding, undersøgelse, korrigerende handlinger og læring.

## Fremgangsmåde
1. **Akut reaktion:** Sikr personer og infrastruktur først. Ved potentielt alvorlig hændelse: stands relevant arbejde/trafik via den ansvarlige rolle og eskalér straks — hurtig eskalering frem for afventning.
2. **Klassificér hændelsen** med **bdk-haendelser-sikkerhedsbrister** som ramme: sikkerhedsmæssig hændelse, administrativ sikkerhedsbrist, eller alvorlig hændelse med undersøgelsesbehov.
3. **Indmeld** ad den korrekte kanal ifølge klassifikationen; notér tid, sted, involverede roller og umiddelbare fakta. Skeln klart mellem fakta og antagelser; undgå skyldplacering uden dokumenteret grundlag.
4. **Undersøg:** fastlæg hændelsesforløb og medvirkende faktorer; inddrag de ansvarlige funktioner (fx sikkerhedskoordinator, sikkerhedsbyggeleder, beredskabsroller) jf. deres funktionsbeskrivelser.
5. **Korrigerende handlinger:** definér tiltag med ansvarlig og deadline; opdatér risikoregisteret jf. **bbtr-risiko-myndighed** og relevante jobinstruktioner jf. **bbtr-produktionssetup**.
6. **Dokumentér** hele forløbet sporbart efter **bbtr-dokumentstyring**, så det kan indgå i projektets sikkerhedsdokumentation og eventuel myndighedsopfølgning.
7. **Lær:** registrér lessons learned og implementér forbedringer versionsstyret via **bbtr-forbedringsloop**; vurdér om hændelsen skal indgå i sikkerhedsrapportering (se **bdk-overvaagning-rapportering**).
