# BDK Overvågning og Rapportering

## Hvornår skillen bruges
- Når brugeren spørger til overvågning af jernbanesikkerheden.
- Når brugeren vil have hjælp til sikkerhedsindikatorer, trends eller kvartals-/årsrapportering.
- Når brugeren vil koble overvågningsfund til risikoejer og risikoprofil.

## Arbejdsgang
1. Læs `references/source-map.md`.
2. Afgræns overvågningsopgaven:
   - indikatoropfølgning
   - trendvurdering
   - kvartals-/årsrapport
   - input til chef- eller direktionsmøder
3. Kortlæg datakilder og signaler:
   - SafetyNet-data
   - audit/tilsynsinput
   - øvrige nævnte datakilder (fx SAP, RDS, Danbro)
4. Kobl fund til:
   - relevant risiko i risikoprofilen
   - risikoejer
   - behov for tiltag/opfølgning
5. Lever en prioriteret rapportstruktur med tydelige næste handlinger.

## Outputformat
1. Formål og afgrænsning
2. Centrale observationer/trends
3. Risikokobling og risikoejerperspektiv
4. Anbefalede handlinger
5. Kilder (repo-stier)
6. Usikkerheder og datagab

## Styringsregler
- Skeln tydeligt mellem observation, vurdering og anbefaling.
- Opfind ikke tal eller KPI-niveauer, som ikke fremgår af kilder.
- Markér datagab eksplicit før konklusioner med høj konsekvens.
