# IT-systemer

## Formål
At understøtte arbejde med Banedanmarks IT- og fagsystemer i drift- og projektsammenhæng: systemadministration, adgangsstyring, teknisk systemansvar og systemintegration — med fokus på sporbarhed, forsvarlig adgangshåndtering og korrekt ansvarsplacering mellem systemadministrator, teknisk systemansvarlig og arkitektroller.

## Fremgangsmåde
1. Identificér hvilket system opgaven vedrører (fx TMS inkl. håndholdte terminaler, asset- og dokumentationssystemer, SharePoint, tekniske systemer i DTC) og hvilken rolle der har ansvaret.
2. Slå rollens ansvar og beføjelser op i den relevante FB — fx "TMS Systemadministrator.pdf" (formål: forsvarlig håndtering af adgange til TMS og TMS-relaterede enheder) og "Systemadministrator for Infrastruktur.pdf" (systemadministration af asset- og dokumentationssystemer). Brug **bdk-roller-kompetencer** til struktureret uddrag.
3. Kortlæg adgangsstyringskrav: hvem tildeler/fjerner adgange, hvordan dokumenteres tildelinger, og hvilken godkendelseskæde gælder. Behandl adgangslister som følsomme data — følg **bdk-gdpr-praksis** for persondata og dataminimering.
4. Ved system- eller grænsefladeændringer: afklar teknisk systemansvar (Teknisk systemansvarlig / Leder af systemansvar) og integrationsansvar (System Integration Architect, System Interface Manager) før ændringen planlægges; ændringer i sikkerhedsbærende systemer følger ledelsessystemets processer (**bdk-trace-processer**).
5. Dokumentér konfigurationer og ændringer versioneret jf. **bbtr-dokumentstyring**; opdater systemdokumentation i samme ændring som systemet.
6. Ved sikkerhedshændelser i IT-systemer (uautoriseret adgang, brud på adgangsstyring): klassificér og håndtér via **bdk-haendelser-sikkerhedsbrister**.
