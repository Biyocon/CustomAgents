# BDK Audit Findings

## Hvornår skillen bruges
- Når brugeren vil planlægge audit, forberede auditinput eller håndtere findings.
- Når brugeren spørger til afvigelser, observationer, forbedringsområder eller opfølgningsfrister.

## Arbejdsgang
1. Læs `references/source-map.md` og vælg audit-type:
   - intern audit
   - leverandøraudit
   - 3.-parts ISO 27001 audit
   - sikkerhedstilsyn i sporet
2. Identificér styringskæden:
   - scope og deltagere
   - registreringssystem (`SafetyNet`, `F2`, `SharePoint` hvor relevant)
   - ansvarlig for opfølgning
3. Klassificér finding-type og krævet håndtering.
4. Udarbejd handlingsplan med frister, dokumentationskrav og eskalationspunkter.

## Outputformat
1. Kontekst og audit-type
2. Findings-klassifikation
3. Handlingsplan (hvem, hvad, hvornar)
4. Dokumentation og systemplacering
5. Kilder (repo-stier)

## Styringsregler
- Hold formuleringer neutrale og kildebaserede.
- Nedgrader ikke obligatoriske krav.
- Markér hvis vejledning konflikter med formel proces- eller lovtekst.
