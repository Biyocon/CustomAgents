# Retfærdiggørelse af Interface Manager-rollen i BBTR

## Formål
At dokumentere og kommunikere den konkrete værdi, som Interface Manager-rollen skaber i krydsfeltet mellem tekniske fagdiscipliner, digitale platforme og organisatorisk governance.

## Rollens værdikæde

### 1. Tværfaglig koordinering
- Sikrer sammenhæng mellem 17 fagposter og de tilhørende fagdiscipliner
- Håndterer grænseflader proaktivt → minimerer fejl i udbudsmateriale
- Koordinerer mellem Spor, Jord, Afvanding, Føringsveje, Kørestrøm, Sikring m.fl.
- **Evidens:** Uden dedikeret koordinering mangler der systematisk håndtering af tværfaglige afhængigheder

### 2. Platform- og datakvalitet
- Ejer kvalitetssikring af BKP v17 datamodel (14 obligatoriske felter, 8 valideringsregler)
- Sikrer konsistens i YB, TBL og bilagsstruktur
- Håndterer support og fejlrettelser (dokumenteret: 5 aktive sager, 1 kritisk)
- **Evidens:** SUP-002 (kritisk) kræver 1 uges manuelt arbejde uden systemrettelse

### 3. Løbende forbedring og udvikling
- Driver udviklingsbacklog (~79 emner totalt, heraf 28 BBTR-specifikke, 16 med højeste prioritet)
- Prioriterer og koordinerer med ekstern leverandør (Fellowmind)
- Kvalitetssikrer leverancer og implementeringer
- **Evidens:** Dokumenteret backlog viser konkret og voksende udviklingsbehov

### 4. Compliance og governance
- CSM-RA og TSI-regulatorisk compliance
- Dokumentstyring med navngivningskonventioner og versionering
- Kvalitetskontrol og Definition of Done for leverancer
- **Evidens:** Banedanmarks tilsynsforpligtelser kræver systematisk kvalitetssikring

### 5. Videndeling og oplæring
- Udarbejdet BBTR Guide (v0.1) til platformens brugere
- Opbygget 14 domænespecifikke videnskills
- Etableret mødestruktur med ledelse (månedlig/kvartalsvis)
- **Evidens:** Guide og skills eksisterer som konkrete leverancer

## Argumentationsstrategi overfor ledelse

### For teamleder (Ahmed)
- Fokus: Daglig operationel værdi, aflastning, fejlhåndtering
- Nøgletal: Antal sager håndteret, timer sparet, fejl forhindret
- Budskab: "Uden denne rolle falder opgaverne mellem stolene"

### For områdechef
- Fokus: Strategisk værdi, risikoreduktion, governance
- Nøgletal: Backlog-størrelse, kritiske fejl, compliance-risici
- Budskab: "Rollen sikrer sammenhæng og kvalitet på tværs"

### For direktion
- Fokus: ROI, risiko, forretningskontinuitet
- Nøgletal: Cost of inaction, udbudsrisiko, tilsynskompliance
- Budskab: "BBTR er forretningskritisk – Interface Manager er den der holder det sammen"

## Forventede kritiske spørgsmål og svar

1. **"Kan opgaven ikke fordeles på eksisterende medarbejdere?"**
   → Nej – opgaven kræver dedikeret tværfaglig koordinering, teknisk platformskendskab OG leverandørstyring. Ingen eksisterende rolle dækker alle tre.

2. **"Hvad skete der før rollen eksisterede?"**
   → Opgaverne blev håndteret ad hoc, uden systematik. Resultatet var ukoordinerede grænseflader, langsom fejlhåndtering og manglende governance.

3. **"Kan vi måle effekten?"**
   → Ja: Antal supportager håndteret, backlog-reduktion, leverandørstyring formaliseret, guide udarbejdet, mødestruktur etableret.
