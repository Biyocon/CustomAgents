# Test og validering

## Formål
At sikre struktureret planlægning, gennemførelse og dokumentation af test- og valideringsaktiviteter i BBTR-projekter, så leverancer valideres mod krav før ibrugtagning, og så jernbanesikkerheden er entydigt forankret under testkørsler.

## Fremgangsmåde
1. Afklar testomfang og testtyper ud fra projektets leverancer (infrastruktur, regler, uddannelse og togkørsel i kombination) — jf. rammerne beskrevet i funktionsbeskrivelsen for Prøvekørselskoordinator.
2. Udarbejd en testplan med ansvarlige roller: Test Manager på projekt-/programniveau planlægger og sikrer ressourcer (Banedanmark, operatører og leverandører); Prøvekørselskoordinator/Testsikkerhedsleder (engelsk: Test Safety Manager) er øverst ansvarlig for jernbanesikkerheden under testkørsler.
3. Afgræns testperioden formelt: ansvaret for testområdet overleveres fra stationsbestyreren til prøvekørselskoordinatoren ved teststart og leveres tilbage ved testafslutning — dokumentér overleveringen.
4. Gennemfør go/no-go-vurdering før teststart ud fra et rent sikkerhedsmæssigt perspektiv; prøvekørselskoordinatoren deltager i beslutningen.
5. Gennemfør tests, registrér afvigelser/defects og koordinér fejlretning; kvalitetssikr testresultater mod acceptkriterier (kobl til bbtr-kvalitet-dod for Definition of Done).
6. Dokumentér valideringsresultater og restpunkter som input til ibrugtagningsprocessen (se bbtr-ibrugtagning); arkivér testdokumentation efter bbtr-dokumentstyring.
7. Før lessons learned tilbage i forbedringsloopet (bbtr-forbedringsloop).
