# Projektstyring

## Formål
At sikre struktureret styring af BaneByg-projekter, så projektet leverer de aftalte leverancer inden for rammerne for tid, omkostninger, kvalitet, omfang og risici, og så den daglige styring foregår inden for styregruppens rammer og Banedanmarks ledelsessystem.

## Fremgangsmåde
1. Fastlæg projektets styringsgrundlag: leverancer, milepæle, budgetramme og succeskriterier; afstem med styregruppe/projektforum og relevante interessenter.
2. Etabler en projektplan med milepæle, afhængigheder og kritisk vej; genbrug produktionssetup-standarderne fra **bbtr-produktionssetup** (projektplan, jobinstruktioner, mødestruktur).
3. Kortlæg leverancer pr. fase og fagpakke via **bbtr-faseopdelt-ydelser** og **bbtr-leverance-mapping**, og tildel ansvarlige og deadlines.
4. Etabler løbende fremdriftsopfølgning: tavlemøder, milestone-tracking og afvigelseshåndtering; eskalér afvigelser ud over tolerancerne til styregruppen.
5. Integrér risikostyring via **bbtr-risiko-myndighed** og hold risikoregisteret opdateret som del af den løbende styring.
6. Rapportér status (fremdrift, økonomi, risici, ændringer) i faste kadencer; ved forretningsprojekter i Banedanmark-regi, se **bdk-projektrapportering-frister** og **bdk-styregruppearbejde**.
7. Dokumentér beslutninger og ændringer versionsstyret jf. **bbtr-dokumentstyring**, og før erfaringer tilbage via forbedringsloopet (**bbtr-forbedringsloop**).
