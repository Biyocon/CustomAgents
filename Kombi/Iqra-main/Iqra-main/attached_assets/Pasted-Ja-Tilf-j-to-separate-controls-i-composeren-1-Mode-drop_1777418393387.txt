Ja. Tilføj to separate controls i composeren:

1. Mode dropdown
   Values:
   - Instant
   - Fast
   - Expert
   - Heavy
   - Auto

2. Conversation Type dropdown
   Values:
   - Chat
   - Agent
   - Research
   - Build

De skal være separate UI-controls og separat state. De må ikke blandes sammen.

Mode dropdown styrer reasoning/compute-depth.
Conversation Type styrer workflow-kontekst.

Implementér override-reglerne præcist:

State:

lastConversationTypeSetBy:
  | "user"
  | "team-mode"
  | "deep-research"
  | "slash-command"
  | null

lastModeSetBy:
  | "user"
  | "deep-research"
  | "slash-command"
  | null

Regler:

- Når brugeren manuelt vælger Conversation Type via dropdown:
  lastConversationTypeSetBy = "user"

- Når brugeren manuelt vælger Mode via dropdown:
  lastModeSetBy = "user"

- Team Mode må autosætte Conversation Type til "Agent", men kun hvis:
  lastConversationTypeSetBy !== "user"

- Når Team Mode autosætter:
  lastConversationTypeSetBy = "team-mode"

- Deep Research må autosætte Conversation Type til "Research", men kun hvis:
  lastConversationTypeSetBy !== "user"

- Deep Research må autosætte Mode til "Expert", men kun hvis:
  lastModeSetBy !== "user"

- Når Deep Research autosætter:
  lastConversationTypeSetBy = "deep-research"
  lastModeSetBy = "deep-research"

- Slash commands må også sætte mode/type:
  /agent → Conversation Type = "Agent"
  /research-agent → Conversation Type = "Research"
  /build → Conversation Type = "Build"
  /expert → Mode = "Expert"
  /instant → Mode = "Instant"

- Når slash command sætter Conversation Type:
  lastConversationTypeSetBy = "slash-command"

- Når slash command sætter Mode:
  lastModeSetBy = "slash-command"

Conflict rule:
- Brugerens manuelle valg vinder altid.
- Hvis brugeren ikke har valgt manuelt, vinder den senest aktiverede capability.
- Team Mode sætter Agent.
- Deep Research sætter Research.
- Hvis både Team Mode og Deep Research er aktive, og ingen user override findes, bestemmer den senest aktiverede capability Conversation Type.

UI:
- Begge dropdowns skal ligge i composer toolbar.
- Mode kan default være "Instant".
- Conversation Type kan default være "Build", hvis placeholderen er “What do you want to build today?”, ellers "Chat".
- Dropdown-valg skal lukke popover og refokusere textarea.
- Dropdowns skal følge single-popover-reglen via activePopover:
  activePopover = "mode"
  activePopover = "conversation"

Vigtigt:
Implementér ikke bare visuelle dropdowns. De skal indgå i normalized submit payload:

{
  mode,
  conversationType,
  ...
}

Og de skal respektere override-reglerne ovenfor.
