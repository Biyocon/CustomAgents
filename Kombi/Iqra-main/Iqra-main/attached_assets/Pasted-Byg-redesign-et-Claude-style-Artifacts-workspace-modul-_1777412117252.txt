Byg/redesign et Claude-style “Artifacts” workspace-modul til OmniAI.

Målet er at implementere tre centrale views:
1. Artifacts Dashboard
2. New Artifact Template Selector
3. Split-screen Artifact Editor med chat til venstre og live preview til højre

Brug eksisterende OmniAI shell, routing, tokens og komponentstil, men redesign modulet så det føles minimalistisk, editorialt og Claude-inspireret.

---

## 1. Artifacts Dashboard

Route:
/artifacts

Layout:
- Full-width workspace view med stor venstremargin og top-padding.
- Øverst:
  - Venstre: stor serif heading “Artifacts”
  - Højre: sort primary button “New artifact”
- Under heading:
  - Section label: “Your artifacts”
  - Kort sort underline under section label
  - Tynd divider line på tværs af container

Grid:
- 3 kolonner desktop
- 2 kolonner tablet
- 1 kolonne mobil
- Card gap ca. 32px horizontal og 36px vertical

Artifact card:
- Preview container:
  - Rounded rectangle
  - Border: light gray
  - Background: off-white / #fafafa
  - Fixed aspect ratio ca. 1.55:1
  - Indeni preview vises et mindre dokumentvindue centreret
  - Dokumentvindue har:
    - hvid baggrund
    - border
    - radius 10–14px
    - clipped overflow
    - enten dokumenttekst, kode snippet eller placeholder-icon
- Under preview:
  - Titel i sans-serif, 18–20px
  - Ellipsis/truncate efter én linje
  - Metadata under titel: “Last edited for X måneder siden”
  - Metadata i grå tekst

Card interaction:
- Hover: subtle shadow, border darkens svagt, scale 1.01
- Click: åbner /artifacts/:id
- Cursor pointer

Eksempeldata:
Lav mindst 12 mock artifacts:
- Code artifacts med TypeScript snippets
- Dokument-artifacts med serif preview
- System design artifacts
- Business docs / PRD
- AI prompt/workflow artifacts
- Placeholder artifact med atom/icon

## 3. Artifact Editor

Route:
/artifacts/:id

Byg split-screen editor i stil med Claude Artifacts.

Overordnet layout:
- Hele viewporten deles i to kolonner:
  - Venstre: chat panel
  - Højre: artifact preview panel
- Lodret resizable divider mellem panelerne
- Begge paneler har egen scroll
- Editor skal kunne vise både dokument-artifacts og UI/system-artifacts

Topbar:
Venstre:
- Artifact title med dropdown chevron

Højre:
- Copy button
- Dropdown chevron ved copy
- Publish button
- Close/X icon
- Preview/code toggle:
  - eye icon = preview
  - code icon = source/code

## 5. Højre Preview Panel — Document Mode

Når artifact type er document:
- Renderer som editorial article
- Hvid baggrund
- Centered article width ca. 720–820px
- Serif typography til dokumentindhold

Document styling:
- H1: 24–30px, bold serif
- Paragraphs: readable line-height 1.55–1.7
- H2/H3 med tydelig spacing
- Inline badges/citations:
  - små rounded badges, light gray
  - fx “Fluence +2”, “LocalLlm +2”
- Tables:
  - thin borders
  - compact rows
  - header row
- Sections med generous vertical spacing

Preview toolbar:
- Sticky top
- Copy
- Publish
- Preview/code toggle

## 8. State Model

Implementér state med Zustand eller lokal React state.

Artifact schema:

{
  id: string
  title: string
  type: "document" | "code" | "system-ui" | "prompt" | "business"
  template: string
  content: string
  sourceCode?: string
  lastEdited: string
  status: "draft" | "published"
  previewKind: "document" | "ui" | "code"
}

Editor state:

{
  currentArtifactId: string
  messages: ChatMessage[]
  selectedModel: "Opus 4.6" | "Sonnet 4.5"
  previewMode: "preview" | "code"
  isPublishing: boolean
  isUsageLimited: boolean
}

ChatMessage schema:

{
  id: string
  role: "user" | "assistant" | "system"
  content: string
  type?: "text" | "progress" | "error" | "file"
}

## 9. Functional Requirements

Dashboard:
- Viser mindst 12 artifacts
- “View more” button nederst
- New artifact navigerer til template selector

Template selector:
- Viser alle 7 template categories
- Klik på template opretter artifact
- Navigerer til editor

Editor:
- Split-screen layout virker
- Chat og preview scroller uafhængigt
- Preview/code toggle virker
- Copy kopierer relevant content/source
- Publish skifter status til published
- Retry button findes ved error state
- Model selector kan skifte model
- File block kan vises i chat

Responsive:
- Dashboard grid: 1/2/3 kolonner
- Template grid: 1/2/4 kolonner
- Editor:
  - Desktop: split view
  - Tablet/mobile: stacked eller tabbed chat/preview

## 10. Design Direction

Visuel stil:
- Minimalistisk
- Claude-inspired
- Editorial
- Høj whitespace
- Subtile borders
- Sort/hvid/grå palette
- Serif headings
- Sans-serif UI labels
- Rounded cards 12–16px
- Ingen tunge farver på dashboard
- Mørk accent kun i system/UI preview og primary buttons

Undgå:
- Overdesign
- For mange farver
- Dashboard chrome
- Tunge gradients
- Komplekse animations
- Inline styles hvis tokens/classes findes

Acceptance criteria:
- UI matcher visuelt de viste Artifacts screenshots.
- Dashboard, template selector og editor er implementeret som selvstændige routes.
- Artifacts kan åbnes fra dashboard.
- Nye artifacts kan startes fra template selector.
- Editor understøtter både document preview og system UI preview.
- Preview/code/publish/copy flows virker.
