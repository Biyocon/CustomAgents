# docs/

> Dokumentation, standarder, projekteksempler og specs.
> Indeholder fagligt referencemateriale, som agenten kan konsultere.

---

## Formål

Denne mappe indeholder **faglig referencemateriale**, som agenten kan bruge til at forstå kontekst, verificere standarder og lære af tidligere projekter. Indholdet er statisk reference, ikke dynamisk arbejdsdokumentation.

---

## Hvilke filer mappen indeholder

| Mappe | Indhold |
|-------|---------|
| `standards/` | Gældende standarder og normer (HD 60364, EN 61439, etc.) |
| `ramboll/` | Rambøll-specifik dokumentation, procedurer, skabeloner |
| `project-examples/` | Anonymiserede eksempler fra tidligere projekter |
| `superpowers/specs/` | Design-specifikationer for nye funktioner |

---

## Hvordan agenten skal bruge mappen

1. Ved tvivl om standarder: konsulter `standards/`.
2. Ved tvivl om Rambøll-praksis: konsulter `ramboll/`.
3. Ved behov for inspiration: konsulter `project-examples/`.
4. Dokumentér, hvilke kilder der er brugt.
5. Opdater ikke referencematerialet uden eksplicit tilladelse.

---

## Hvad der ikke må gemmes her

- Projekt-specifikke leverancer
- Fortrolige eller personfølsomme oplysninger
- Kontraktkritiske data
- Uautoriserede kopier af standarder

---

## Eksempel på brug

> Agent skal verificere en kabelberegning:
> 1. Konsulterer `docs/standards/` for gældende kabelformler
> 2. Tjekker `docs/ramboll/` for Rambøll-specifik praksis
> 3. Verificerer beregningen
> 4. Citerer kilden i output
